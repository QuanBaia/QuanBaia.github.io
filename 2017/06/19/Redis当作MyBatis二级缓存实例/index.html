发表于 2017-06-19 | 分类于 Java |

Redis 二级缓存

由于 Redis 的极佳表现和可以持久化的操作越来越受企业的欢迎，我们今天就来探讨一下，怎么使用 Redis 当做 MyBatis 的二级缓存

优点

  • 使用缓存可以有效的避免重复性的请求服务器数据库，减少对服务器的请求次数，更好的用户体验
  • Redis相对于其他 Key–Value 数据库，Redis 支持数据 持久化
  • 可以进行数据库备份，即 Master-slave 模式的数据备份
  • 速度方面 Redis 的性能极佳，读的速度是110000次/s,写的速度是81000次/s
  • 丰富的数据类型 – Redis支持二进制案例的 String，List，Hashe，Set 及 Ordered Set 数据类型操作
  • Redis 的所有操作都是原子性的，同时 Redis 还支持对几个操作全并后的原子性执行
  • Redis还支持 publish/subscribe，通知等等

需要准备

在 SpringMVC 中加入对 Redis 提供的 Jar 包:

  • commons-pool2-2.4.2.jar (使用 Redis 的连接池)
  • jedis-2.4.2.jar (Java 对于 Redis 的开发包)
  • spring-data-redis-1.6.0.RELEASE.jar (高度封装 jedis,操作redis中的数据)

VMware Workstation Pro 虚拟机，在 Linux 下安装 Redis

RedisDesktopManager (Windos下的Redis视图化操作软件 )

在 Linux 下部署 Redis

首先我们需要把 Redis 安装

Ubuntu

如果你是 Ubuntu 下的用户，安装就比较方便

首先我们要打开终端进行操作,进行下载安装

  $sudo apt-get install redis-server

启动 Redis 服务

  $redis-server

看到这个图片说明已经成功安装

客户端连接 Redis

  $redis-cli

进入操作界面，任何我们 Ping 一下试试能不能 Ping 通

  127.0.0.1:6379>ping
  PONG

表示已经成功连接

另一种安装方式

我们需要把 Redis 安装包下载下来，自行安装

  $ wget http://download.redis.io/releases/redis-3.2.9.tar.gz
$ tar xzf redis-3.2.9.tar.gz
$ cd redis-3.2.9
$ make

编译完成之后，在 src 目录文件下我们可以找到 4 个可执行的文件redis-server、redis-benchmark、redis-cli和redis.conf。我们把他们拷贝到一个目录下，这样我们就可以方便的配置和管理我们的 Redis

  mkdir /usr/redis
cp redis-server /usr/redis
cp redis-benchmark /usr/redis
cp redis-cli /usr/redis
cp redis.conf /usr/redis
cd /usr/redis

  • 这个 redis.conf 是 Reids 的配置文件，在里面可以设置密码等等，在启动的时候我们只需要指定一下配置文件就可以了

    这样基本上就是安装完成了，测试部分和 Ubuntu 的步骤一样

配置 Redis 文件

因为我们是用 Windos 来连接 Redis 所以要稍微修改一下配置文件

我们把这一块要注释掉，就是不绑定主机，如果绑定了，别人就不能连接了

找到这一块，添加一个密码，不然我们会报错发现会报 DENIED Redis is running in protected mode because protected mode is enabled…UI 异常，就是说我们的 Reids 现在属于受保护的状态，我们设置密码以后就可以解决这个问题

设置完成之后我们需要指定该配置文件进行启动 Reids 服务

  $redis-server /usr/redis/reids.conf

启动之后我们发现我们连接客户端时不输入密码也可以进入，但是我们不能进行任何操作，我们需要用密码登入才可以

  $redis-cli -a myredis

-h 是连接的主机ip，host的缩写，-p 是端口 port的缩写 ，-a 后面是密码（requirepass 后面配置的）auth的缩写

我们测试一下，如果ping通过，那么在Linux下的部署就已经完成了

然后输入对应的信息，name–》创建名，host–》ip地址，port–》端口号，auth–》密码

点击测试连接，如果提示成功的话，就成功了，如果没有成功，我们仔细查看 Linux 的 Ip 地址以免输入错误，再不对的话，仔细检查上面步骤
